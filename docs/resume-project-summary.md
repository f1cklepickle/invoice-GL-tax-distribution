# Resume Project Summary

## Project Name

Invoice GL Tax Distribution

## Short Project Description

Built a browser-based invoice coding helper that groups line items by GL code,
calculates subtotals and tax difference, distributes tax across GL groups, and
generates a copyable coded invoice summary for review.

## Honest Scope Notes

- Personal workflow and portfolio project.
- Static HTML, CSS, and JavaScript app.
- Uses fake/demo invoice data in the repository.
- Uses browser-local preferences and shortcuts.
- Not an official employer system.
- Not integrated with accounting, ERP, AP automation, or approval software.
- Saved invoice records and import/export are planned but not implemented yet.

## Resume Bullet Options

- Built a browser-based invoice coding helper in HTML, CSS, and JavaScript to
  group invoice line items by GL code and calculate GL-level tax distribution.

- Designed a small AP workflow tool that supports invoice line-item review,
  refund handling, tax/difference calculation, and copyable coded summaries.

- Added browser-local settings, profile preferences, GL shortcuts, and item-name
  autocomplete to make repeated invoice entry faster and more consistent.

- Created calculation checks for invoice subtotal, refund handling, tax
  distribution, and rounding behavior so GL totals tie back to invoice totals.

- Developed a fake demo invoice workflow to show invoice coding behavior without
  adding private vendor, employer, invoice, or GL data.

## Portfolio Summary Option

Invoice GL Tax Distribution is a practical frontend project based on accounts
payable invoice review work. It demonstrates how a small browser tool can help
organize invoice line items, group costs by GL code, distribute tax across those
groups, and produce a coded summary for review. The project is intentionally
scoped as a personal workflow aid and portfolio example, not an official
accounting system.

## LinkedIn Or Project Blurb Option

I built a small browser-based invoice coding helper to practice frontend
development while modeling a real AP review workflow. The app accepts invoice
line items, groups them by GL code, calculates tax distribution, and generates a
copyable coding summary. It uses fake demo data and browser-local preferences,
with saved invoice storage and import/export planned as future improvements.

## Wording To Avoid

Avoid wording that implies the app is broader than it is:

- Do not say it is company-wide.
- Do not say it is production-grade.
- Do not say it is an official accounting system.
- Do not say it automates AP end to end.
- Do not claim adoption, approval, savings, or error reduction without evidence.
